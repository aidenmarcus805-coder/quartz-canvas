Executive Summary
Hercules 26B V2.5 is the third-axis upgrade to Hercules V2.0. Whereas Hercules V2.0 installed routing, diffusion, and
speculative decoding, V2.5 opens a fundamentally new dimension of compute: Tensor Product Attention gives the model a third
relational axis so tokens can attend to the interaction between other tokens, not just the tokens themselves. That third axis then
unlocks an entire cascade of follow-on innovations — Clifford Geometric Attention, Relational Test-Time Training, TD-MoE
tensor expert compression, TT-LoRA MoE fine-tuning, Structural Token Merging, Manifold-Constrained Block Diffusion, and
Tensorial RAG — all of which are architecturally impossible in a bilinear transformer. The 26B A4B is better suited to many of
these ideas than the 31B Dense because its native expert bank is the exact structure that TD-MoE, TT-LoRA MoE, and
relational expert routing were designed for. Hercules 26B V2.5 is the faster-shipping, production-first Hercules. The 31B V2.0 is
the bigger endgame.

All 15 Innovations — Technical Detail
① Tensor Product Attention — Phase 1 First Install [THIRD AXIS]
Replace GQA-style attention on global layers with TPA. Q=Qa⊗Qb, K=Ka⊗Kb. Token i attends to the interaction between
tokens j and k. The relational third axis opens. KV cache shrinks because factors are smaller. No production open-source
model has shipped TPA.
→ Directly upgrades the 26B A4B global attention layers
 
→ Foundation for every later tensor-based idea
 
→ Implementable from ICML/NeurIPS 2025 open-source code
 
② Clifford Geometric Attention — Direction-Aware Relational Reasoning
Extends TPA by using the full Clifford geometric product instead of just the tensor outer product. The bivector component
encodes the orientation of the plane spanned by two token vectors. The model gets a native sense of "which direction does
this logical relationship flow?" — essential for causal reasoning, code call-graphs, and proof chains.
→ Linear complexity via CliffordNet January 2026 architecture
 
→ Token-pair direction encoded, not just strength of relation
 
→ Swap for TPA in the most reasoning-heavy layers
 
③ TD-MoE — Tensor Decomposition for the Native Expert Bank [BEST FIT FOR 26B]
Stack all expert weight matrices into a 3D tensor, apply multi-linear whitening to remove cross-expert redundancy, then
Tucker-decompose with ranks r1/r2/r3. Achieves massive expert bank compression by factoring out the shared subspace
across experts. The 26B A4B native expert bank is the exact target structure this was designed for — better fit than on the
31B.
→ No training required; plug-in expert compression
 
→ Expert bank VRAM collapses significantly
 
→ Potentially improves reasoning quality by denoising weight structure
 
④ TT-LoRA MoE — Tensorized Per-Expert Fine-Tuning Adapters
Replace flat QLoRA adapters with tensor-train LoRA adapters per expert with a sparse router that activates only relevant
adapters. Reported at 0.03% of AdapterFusion parameters while outperforming it on multi-task benchmarks. Every fine-tuning
run — Symphony-MoE injection, Opus 4.6 distil, domain specialization — gets a quadratically larger hypothesis space at
nearly the same adapter cost.
→ Much more expressive adapters at same parameter budget
 
→ Specialist forks fine-tuned more cleanly
 
→ Used for all Phase 2+ fine-tuning steps
 
⑤ Relational Expert Routing — Route by Structural Signature
Once TPA is installed, each token has a tensor interaction signature in context. Route experts using that signature as an
additional routing signal instead of relying purely on soft domain tags. Code tokens with function-call relational structure route
to code experts. Proof tokens with entailment relational structure route to reasoning experts. No domain tag needed.
→ Domain-agnostic structural routing
 
→ More discriminative than flat embedding routing
 
→ Direct upgrade to existing Routing-Free MoE

⑥ Tensor Conceptors — Hard Geometric Structural Constraint
Conceptors define ellipsoidal regions in activation space. Tensor Conceptors extend this to the tensor product manifold.
Define a Code Tensor Conceptor (valid code dependency structures) and a Reasoning Tensor Conceptor (valid logical
entailment structures). Project third-axis computation to stay inside the relevant conceptor at every attention step. The model
cannot structurally violate code logic or reasoning chains.
→ Hard structural enforcement — not RLHF, not a verifier
 
→ Architecture-level logical consistency guarantee
 
→ Especially powerful for Quartz Editor video-editing logic
 
⑦ ODE Tensor Steering — Continuous Relational Alignment
ODESteer treats activation steering as an ODE with barrier functions. Applied to the third axis: steer the relational tensor flow
continuously during generation, not with a blunt single-vector injection. AUSteer atomically selects only the discriminative
tensor dimensions to modify.
→ Context-sensitive multi-layer tensor steering
 
→ 10× more precise than standard block-level activation steering
 
→ No fine-tuning required
 
⑧ Manifold-Constrained Block Diffusion — Geodesic Reasoning
TPA weight factorizations live on the Segre variety — a specific algebraic manifold. Standard Block Diffusion denoises in flat
token embedding space. Constrain denoising to stay on the Segre variety. Every denoising step becomes a geodesic — the
shortest path in relational space from noise to coherent output. Reasoning cannot produce relationally incoherent intermediate
states.
→ Denoising steps are provably structure-consistent
 
→ Shorter geodesics = fewer denoising steps needed
 
→ Fewer hallucinations at intermediate generation states
 
⑨ Structural Token Merging — Compress by Relational Equivalence
Standard ToMe merges tokens with similar content embeddings. Structural Token Merging merges tokens with the same
third-axis tensor signature in context. Two function calls with identical argument-passing structure but different names are
relationally equivalent and can be merged. Compresses 256K context without discarding structural information.
→ 30–50% context compression without structural information loss
 
→ More aggressive than content-based merging
 
→ Critical for long-context serving on consumer hardware
 
⑩ Relational Test-Time Training — Fast-Weight Third Axis Adaptation
TTT-E2E showed test-time weight updates are 35× faster than full attention at 2M context. Targeting only TPA factor matrices
(Qa, Qb, Ka, Kb) instead of full MLPs keeps update cost minimal while adapting exactly the component that encodes relational
structure. The model reads a codebase and physically adapts its relational attention weights to that codebase before
generating any output token.
→ Model internalizes document relational topology before first output
 
→ Transformative for long-context coding tasks like Quartz Editor workflows
 
→ Factor matrices are much smaller than MLP weights — fast to update

■ Routing-Free MoE + Symphony-MoE [V2.0 core, now tensor-enhanced]
Same as V2.0 but now the router receives the third-axis tensor signature as an additional conditioning signal. Symphony-MoE
code and vision specialization stays. Domain tags stay. The whole routing system becomes more discriminative.
→ Best coding benchmark uplift driver
 
→ Domain tag + tensor signature dual routing
 
→ Code fork LiveCodeBench target: 85–92%
 
■ Gated DeltaNet Linear Memory
O(n) linear recurrent attention for long-range layers. Less impactful than on the 31B (5 global layers vs 60) but still essential
for practical 256K context on consumer hardware.
→ Critical for long-context MRCR and LongBench tasks
 
→ Pairs with TurboQuant and Structural Token Merging
 
■ TurboQuant KV + LoRA-Drop + 128-byte Layout
TurboQuant compresses KV 5–6×. LoRA-Drop adds temporal reuse (2.6× decode, 45–55% KV). 128-byte alignment recovers
20–30% memory bandwidth. All three stack multiplicatively.
→ Compound memory and bandwidth savings
 
→ LoRA-Drop is plug-in: zero training cost
 
→ Critical for 12–18 GB deployment tiers
 
■ Tensorial RAG — Retrieve by Structural Similarity
Instead of embedding cosine similarity (bilinear), store document representations as tensor products and retrieve by manifold
distance in the tensor product space. For Quartz Editor: retrieve video clips with the same temporal-causal relational structure
as a reference. For coding: retrieve functions with the same logical call-graph topology.
→ Structural retrieval — possible only with third axis installed
 
→ Transformative for code and video editing workflows
 
→ Long-term competitive differentiator
 
■ Opus 4.6 CoT Distillation
~3,300 Claude Opus 4.6 CoT traces, QLoRA SFT, 1-epoch hard cap. Highest ROI step in the build. Now done with TT-LoRA
MoE adapters for better expressivity.
→ ~$30–80, 4–8 hours on H100
 
→ GPQA +2–4%, AIME +2–3%, HLE +3–4%
 
→ Best run after TPA + Symphony-MoE

HERCULES 26B V2.5 — Quartz Editor internal research designation
Hercules 26B V2.5 is a research architecture specification. Benchmark estimates are projections from stacked published research directions. The
model does not yet exist as a trained checkpoint. TPA (ICML/NeurIPS 2025), TD-MoE (ICLR 2026), TT-LoRA MoE (April 2025), CLaSp (May 2025),
and LoRA-Drop (January 2026) all have open-source implementations. Clifford Geometric Attention, Manifold Block Diffusion, Tensor Conceptors,
and Relational TTT are novel applications of published mathematical frameworks. Produced by Quartz Editor Research · April 2026.
